# Business Problem

E-commerce reporting depends on multiple operational tables: orders, order items, payments, customers, products, sellers, reviews, and delivery timestamps. If those tables are connected directly to a dashboard without profiling, cleaning, modeling, and reconciliation, the business can end up with inconsistent revenue, unclear delivery performance, missing product categories, and unreliable seller or customer reporting.

This project solves that problem by creating a validated reporting layer from the Olist Brazilian E-Commerce Public Dataset.

## Business Scenario

An e-commerce leadership team needs a trusted reporting foundation that can answer:

- How much revenue was generated by month?
- Which product categories drive the most revenue?
- Which customer states generate the most demand?
- Which sellers drive the most sales?
- How often are delivered orders late?
- Where do source data quality issues appear?
- Do Python and SQL transformation paths produce matching results?

## Solution

The project builds the reporting layer two ways:

1. Python/Pandas transforms raw CSV files into curated dimensions, facts, quality logs, and monthly KPIs.
2. SQLite SQL reproduces the same logic from the same raw files.
3. A comparison layer validates the Python and SQL outputs.
4. Charts and diagrams summarize revenue trends, category performance, architecture, data model, and validation results.

## Completed Results

- 99,441 orders modeled.
- 112,650 order item lines modeled.
- 25 monthly KPI rows created.
- 15.84M total revenue calculated as item price plus freight.
- 96,478 delivered orders identified.
- 7,826 late delivered orders identified.
- 2,440 data quality issues logged.
- 14 of 14 Python-vs-SQL metric checks passed.
- 25 of 25 monthly revenue checks passed.

## Portfolio Framing

This is not just a data cleaning project. It is a reporting pipeline and quality control project that demonstrates how raw operational data can become trusted BI-ready data through profiling, transformation, dimensional modeling, validation, and documentation.
