# Data Dictionary

This data dictionary covers the curated reporting outputs created by Project 3. The raw Olist files remain in `data/raw`; the reporting outputs are generated by the Python and SQL pipelines.

## Dimensions

### `dim_customer`

Grain: one row per `customer_id`.

| Column | Description |
| --- | --- |
| `customer_id` | Order-level customer identifier |
| `customer_unique_id` | Unique customer identifier across orders |
| `customer_zip_code_prefix` | Customer zip prefix |
| `customer_city` | Standardized customer city |
| `customer_state` | Standardized customer state |

### `dim_product`

Grain: one row per `product_id`.

| Column | Description |
| --- | --- |
| `product_id` | Product identifier |
| `product_category_name` | Original Portuguese product category |
| `product_category_name_english` | English product category; missing values filled as `unknown` |
| `product_category_label` | Display label for BI reporting |
| `product_name_lenght` | Product name length from source data |
| `product_description_lenght` | Product description length from source data |
| `product_photos_qty` | Number of product photos |
| `product_weight_g` | Product weight in grams |
| `product_length_cm` | Product length in centimeters |
| `product_height_cm` | Product height in centimeters |
| `product_width_cm` | Product width in centimeters |

### `dim_seller`

Grain: one row per `seller_id`.

| Column | Description |
| --- | --- |
| `seller_id` | Seller identifier |
| `seller_zip_code_prefix` | Seller zip prefix |
| `seller_city` | Standardized seller city |
| `seller_state` | Standardized seller state |

## Facts

### `fact_orders`

Grain: one row per `order_id`.

| Column | Description |
| --- | --- |
| `order_id` | Order identifier |
| `customer_id` | Customer identifier joined from orders |
| `customer_unique_id` | Unique customer identifier |
| `customer_city` | Customer city |
| `customer_state` | Customer state |
| `order_status` | Source order status |
| `order_purchase_timestamp` | Purchase timestamp |
| `purchase_date` | Purchase date |
| `purchase_month` | Purchase month in `YYYY-MM` format |
| `order_approved_at` | Approval timestamp |
| `order_delivered_carrier_date` | Carrier delivery timestamp |
| `order_delivered_customer_date` | Customer delivery timestamp |
| `order_estimated_delivery_date` | Estimated delivery timestamp |
| `is_delivered` | Delivered flag |
| `delivery_days` | Days from purchase to customer delivery |
| `estimated_delivery_days` | Days from purchase to estimated delivery |
| `days_late` | Days between actual and estimated delivery |
| `is_late_delivery` | Late delivered order flag |
| `item_price_total` | Sum of item price for the order |
| `freight_total` | Sum of freight value for the order |
| `total_revenue` | Item price total plus freight total |
| `payment_total` | Sum of payment values for the order |
| `payment_order_variance` | Payment total minus total revenue |
| `payment_variance_abs` | Absolute payment/order variance |
| `has_payment_variance_flag` | Flag where absolute variance exceeds 0.05 |
| `order_item_count` | Number of item lines |
| `unique_product_count` | Distinct products in the order |
| `unique_seller_count` | Distinct sellers in the order |
| `missing_product_category_count` | Number of item lines with missing or unknown category |
| `payment_count` | Number of payment rows |
| `payment_methods` | Payment methods used on the order |
| `max_payment_installments` | Maximum installment count on the order |
| `avg_review_score` | Average review score for the order |
| `review_count` | Review count for the order |
| `has_review_comment` | Whether the order has a review comment |
| `average_order_value` | Order-level revenue value; used for summary AOV |

## Summary Tables

### `monthly_kpi_summary`

Grain: one row per purchase month.

| Column | Description |
| --- | --- |
| `purchase_month` | Purchase month |
| `total_revenue` | Total item-plus-freight revenue |
| `item_price_revenue` | Item price revenue |
| `freight_revenue` | Freight revenue |
| `payment_revenue` | Payment revenue |
| `payment_order_variance` | Payment revenue minus item-plus-freight revenue |
| `total_orders` | Distinct orders |
| `delivered_orders` | Delivered orders |
| `late_delivery_count` | Late delivered orders |
| `avg_delivery_days` | Average delivery days |
| `unique_customers` | Distinct unique customers |
| `unique_sellers` | Distinct sellers |
| `average_order_value` | Total revenue divided by total orders |
| `late_delivery_rate` | Late delivery count divided by delivered orders |

### `data_quality_issues`

Grain: one row per data quality issue.

| Column | Description |
| --- | --- |
| `issue_id` | Issue identifier |
| `source_table` | Source table where issue was found |
| `record_id` | Record or summary ID |
| `field_name` | Field connected to the issue |
| `issue_type` | Issue category |
| `severity` | Critical, High, Medium, or Low |
| `issue_description` | Human-readable issue explanation |

### `python_reconciliation_summary` and `sql_reconciliation_summary`

Grain: one row per validation metric.

| Column | Description |
| --- | --- |
| `metric` | Metric name |
| `python_value` / `sql_value` | Metric value from the pipeline path |

### `python_sql_comparison_summary`

Grain: one row per comparison metric.

| Column | Description |
| --- | --- |
| `metric` | Metric name |
| `python_value` | Python metric value |
| `sql_value` | SQL metric value |
| `difference` | Python value minus SQL value |
| `absolute_difference` | Absolute difference |
| `status` | Pass or Review |
