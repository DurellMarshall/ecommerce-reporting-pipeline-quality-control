## What I Understand About Project 3

Project 3 is the technical proof project.

It should show that Durell can take real multi-table commercial data and turn it into a validated reporting layer. The important story is not simply "I cleaned CSV files." The stronger story is:

> I built parallel Python and SQL reporting pipelines from raw operational e-commerce data, reconciled the outputs, documented quality rules, and produced dashboard-ready tables.

Project 3 should prove:

- raw data profiling
- business-question framing
- Python/Pandas transformation
- SQL transformation
- joins across related operational tables
- dimension/fact reporting design
- data quality checks
- reconciliation between two transformation paths
- documentation and reproducibility
- readiness for BI tools

## Project 3 Business Questions

The final reporting layer should answer:

- How much revenue was generated by month?
- How many total orders and delivered orders were recorded?
- What is the average order value?
- Which product categories drive the most revenue?
- Which customer states generate the most demand?
- Which sellers drive the most sales?
- How often are orders delivered late?
- Which categories or sellers have higher delivery risk?
- Where do missing categories, missing timestamps, or reconciliation differences appear?
- Do the Python and SQL outputs agree on the major business metrics?

## Project 3 Validation Checks

The reconciliation layer should compare:

- Python total revenue vs SQL total revenue
- Python order count vs SQL order count
- Python delivered order count vs SQL delivered order count
- Python late delivery count vs SQL late delivery count
- Python missing product category count vs SQL missing product category count
- Python unique customer count vs SQL unique customer count
- Python unique seller count vs SQL unique seller count
- Payment total vs item price plus freight total

1. Expand the profiling script so it records:
   - row counts
   - column counts
   - duplicate row counts
   - missing values by column
   - primary key duplicate checks
   - basic date ranges

   - load raw Olist tables
   - parse date fields
   - normalize category translation
   - build delivery timing fields
   - flag missing/invalid values
3. Build `python/03_create_python_reporting_tables.py`:
   - create dimensions
   - create facts
   - create monthly KPI summary
   - create data quality issue output
4. Build SQL pipeline:
   - `sql/01_create_tables.sql`
